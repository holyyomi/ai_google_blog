# Golden Article Examples

## 0. 최종 포지션과 편집장 기준

최종 포지션은 **AI자동화돈 되는 생활 선택 기준 미디어**다.

주요 독자는 **30~50대 직장인**이며, 글의 톤은 친절한 생활정보형이되 돈, 시간, 생산성, 생활비, 디지털 변화에서 손해 보지 않는 선택 기준을 날카롭게 제시해야 한다.

모든 골든 샘플은 아래 약속을 지킨다.

- 첫 화면에서 이 글이 필요한 사람이 보인다.
- 한 문장 결론이 추상어가 아니라 행동 기준으로 나온다.
- 읽고 나서 바로 할 일이 3개 이상 보인다.
- 표, 체크리스트, FAQ는 장식이 아니라 판단을 돕는다.
- 해시태그는 content_type과 정확히 맞고 다른 유형을 섞지 않는다.

### 0-1. 5개 카테고리 축

- AI 자동화: AI 기능, 업무 자동화, 생산성, 검수 시간 절감.
- 돈 되는 이슈: 지원금, 세금환급, 환급금, 생활비, 수수료, 최종 결제금액.
- 생활 선택 기준: 환불, 소비자 피해, 유행 소비, 오픈런, 품절, 일상 선택.
- 디지털 생존법: 플랫폼 변경, 앱 업데이트, 서비스 종료, 계정, 결제, 백업.
- 연예·스포츠·OTT 이슈 해석: 연예/스포츠/OTT/팬덤/커뮤니티 이슈를 반응 구조와 플랫폼·소비 흐름으로 해석. 조회수 폭발형 트래픽 엔진.

### 0-2. 7개 글 유형

1. 이슈 해석형
   - 목적: 왜 이슈가 퍼졌는지보다 내 선택에 어떤 신호인지 설명한다.
   - 제목 공식: 이슈 + 궁금증 + 판단 기준.
   - 필수 요소: 확산 이유, 숨은 신호, 독자 예시, 오늘 할 일.
   - 금지 패턴: 논란 감상, "사람들이 놓친", "진짜 변수".
   - 해시태그/품질: 트렌드와 선택기준 중심, action guide 필수.

2. 실수 방지형
   - 목적: 환불, 결제, 플랫폼 변경에서 손해를 막는다.
   - 제목 공식: 상황 + 손해 + 먼저 남길 증거/행동.
   - 필수 요소: warning box, 증거 목록, 대응 순서, FAQ.
   - 금지 패턴: 공포 조장, 사건사고 자극.
   - 해시태그/품질: 환불/소비자피해/체크리스트 중심.

3. 비교선택형
   - 목적: 가격, 조건, 옵션을 비교해 최종 선택을 돕는다.
   - 제목 공식: 선택지 + 숨은 비용 + 비교 이득.
   - 필수 요소: 비교표, 계산 예시, 최종 판단 기준.
   - 금지 패턴: 특정 브랜드 홍보, 할인 문구 복붙.
   - 해시태그/품질: 생활비/비용비교/절약정보 중심.

4. 방법론가이드형
   - 목적: 신청, 설정, 조회, 자동화 과정을 바로 실행하게 한다.
   - 제목 공식: 해야 할 일 + 먼저 볼 조건 + 얻는 이득.
   - 필수 요소: 단계, 필요 서류/설정, 공식 확인처, FAQ.
   - 금지 패턴: "확인하세요"만 반복.
   - 해시태그/품질: 타겟 독자와 core_message 필수.

5. 돈수익 분석형
   - 목적: 환급, 세금, 지원금, 생활비 손익을 구분한다.
   - 제목 공식: 돈 키워드 + 조회/대상 + 손해 방지.
   - 필수 요소: 대상, 경로, 금액/시점, 계좌/서류, 주의 오류.
   - 금지 패턴: 세금환급 글에 사용처/바우처/가맹점 혼합.
   - 해시태그/품질: tax_refund와 policy_deadline 해시태그 분리.

6. 큐레이션형
   - 목적: 여러 선택지를 생활 기준으로 골라준다.
   - 제목 공식: 상황 + 고르는 기준 + 바로 쓸 이득.
   - 필수 요소: 선별 기준, 추천 이유, 내부 링크 후보, 다음 행동.
   - 금지 패턴: 기준 없는 나열.
   - 해시태그/품질: 선택기준/생활정보 중심.

7. viral_issue_decode
   - 목적: 연예/스포츠/OTT/커뮤니티 이슈를 조회수형으로 해석하되, 루머가 아니라 반응 구조와 독자 관심 포인트를 정리한다.
   - 제목 공식: [인물/작품/경기] + 반응이 갈린 이유 + 핵심 포인트.
   - 필수 요소: 이슈 요약, 클릭 이유, 반응 포인트 3개, 플랫폼/팬덤/소비 구조 해석, 다음 포인트, evergreen 내부링크 후보, FAQ.
   - 금지 패턴: 루머/사생활 폭로, "충격·소름·난리" 단어.
   - 해시태그/품질: 루머 없음, 공식 기사 기반, 내부링크 후보 필수.

이 문서는 Blogspot 오늘 이슈 자동화가 따라야 할 골든 콘텐츠 기준이다. 자동화는 기사 제목을 그대로 요약하지 않고, 독자의 돈, 시간, 불안, 선택 기준으로 다시 번역한 생활 의사결정 글을 만들어야 한다.

## 1. 좋은 글의 공통 기준

- 제목이 독자의 문제를 바로 건드린다.
- 첫 150자 안에 답이 있다.
- 타겟 독자가 명확하다.
- 핵심 메시지가 한 문장으로 정리된다.
- 본문에 실제 행동 정보가 있다.
- 표, 체크리스트, FAQ가 형식이 아니라 정보 전달에 기여한다.
- 해시태그가 주제와 정확히 맞다.
- 지원금, 환급, 환불, AI, 플랫폼 주제가 서로 섞이지 않는다.

## 2. 나쁜 글의 공통 기준

- "공식 확인하세요"만 반복한다.
- 주제가 넓고 흐리다.
- 지원금과 세금환급 표현이 섞인다.
- 제목은 구체적인데 본문은 일반론이다.
- FAQ가 generic하다.
- 해시태그가 주제와 맞지 않는다.
- 읽고 나서 바로 할 행동이 없다.

## 3. 지원금형 골든 샘플 구조

주제 예: 고유가 피해지원금 신청 전 확인할 것

필수 구조:

1. 누가 봐야 하나
   - 고유가 피해지원금 대상 여부, 신청 기간, 사용처를 빠르게 확인하고 싶은 사람.

2. 이 글의 결론
   - 고유가 피해지원금은 금액보다 대상 조건, 신청 기간, 지급 방식, 사용처를 먼저 확인하는 것이 핵심이다.

3. 정보표
   - 신청 대상
   - 신청 기간
   - 지급 금액
   - 신청 방법
   - 지급 방식
   - 사용처
   - 중복 지원 여부
   - 필요 서류
   - 공식 확인처

4. 신청 전 체크리스트
   - 공식 공고의 사업명과 담당 기관 확인
   - 본인 대상 조건 확인
   - 신청 기간과 접수 시간 확인
   - 요일제 또는 방문 접수 여부 확인
   - 필요 서류 준비
   - 지급 방식 확인
   - 사용처와 제외 업종 확인
   - 중복 지원 제한 확인
   - 신청 완료 화면 보관

5. 놓치기 쉬운 함정
   - 비슷한 이름의 지원사업 혼동
   - 신청 기간 착각
   - 접수 시간 또는 요일제 누락
   - 사용처 제한 미확인
   - 서류 발급 지연

6. 주제명 기반 FAQ
   - 고유가 피해지원금 신청 대상은 누구인가요?
   - 고유가 피해지원금 신청 기간은 언제까지인가요?
   - 고유가 피해지원금은 어디에서 사용할 수 있나요?

7. 해시태그 예시
   - #고유가피해지원금 #지원금신청 #신청기간 #대상조건 #사용처 #정부지원 #생활정보

## 4. 세금환급형 골든 샘플 구조

주제 예: 세금 환급금 조회 전 홈택스에서 먼저 볼 3가지

필수 구조:

1. 누가 봐야 하나
   - 세금 환급 대상 여부, 조회 경로, 환급 계좌와 필요 서류를 먼저 확인하고 싶은 사람.

2. 이 글의 결론
   - 세금 환급은 내가 환급 대상인지, 어디서 조회·신청해야 하는지, 환급 계좌와 필요 서류가 맞는지 확인하는 것이 핵심이다.

3. 정보표
   - 환급 대상
   - 조회 경로
   - 신고 경로
   - 홈택스/손택스 여부
   - 필요 서류
   - 환급 계좌
   - 지급 예상 시점
   - 주의할 오류
   - 공식 확인처

4. 홈택스/손택스 확인 순서
   - 홈택스 또는 손택스 로그인 가능 여부 확인
   - 본인 인증 수단 확인
   - 환급금 조회 메뉴 확인
   - 신고 내역과 납부 내역 대조
   - 환급 계좌 정보 확인
   - 필요 서류와 보완 요청 여부 확인
   - 접수 번호 또는 조회 화면 보관

5. 환급 지연 원인
   - 계좌번호 오입력
   - 예금주 정보 불일치
   - 공제 자료 누락
   - 중복 신고
   - 연락처 오류
   - 보완 요청 미확인

6. 주제명 기반 FAQ
   - 세금 환급 대상은 어떻게 확인하나요?
   - 세금 환급은 어디에서 조회하나요?
   - 세금 환급 신청 전에 어떤 정보를 준비해야 하나요?

7. 해시태그 예시
   - #세금환급 #환급금조회 #홈택스 #손택스 #국세환급금 #환급계좌 #생활정보

세금환급형 금지 표현:

- 사용처
- 바우처
- 가맹점
- 지역상품권
- 지급 방식 단정

단, 실제 공식 정보에 해당 지급 방식이 명시된 경우만 예외적으로 허용한다.

## 5. 환불/소비자 피해형 골든 샘플 구조

주제 예: 환불 지연 때 소비자가 먼저 남겨야 할 증거

필수 구조:

1. 먼저 남길 증거
   - 결제내역
   - 주문번호
   - 상품 페이지
   - 배송 상태
   - 고객센터 상담 기록
   - 판매자 답변
   - 환불 예정일 안내

2. 대응 순서
   - 판매자에게 같은 내용으로 문의
   - 플랫폼 고객센터에 접수
   - 카드사 또는 결제수단 고객센터 확인
   - 접수 번호와 날짜 기록
   - 답변 지연 시 이전 기록을 묶어 재문의

3. 환불 지연 시 체크리스트
   - 결제 승인일 확인
   - 주문번호 캡처
   - 고객센터 접수 번호 기록
   - 환불 예정일 확인
   - 카드사 승인 취소 여부 확인
   - 판매자 답변 보관
   - 플랫폼 정책 페이지 확인

4. FAQ
   - 환불이 늦어질 때 가장 먼저 남길 증거는 무엇인가요?
   - 고객센터 답변이 늦으면 어떻게 해야 하나요?
   - 결제 취소가 안 되면 카드사에 무엇을 확인해야 하나요?

## 6. 플랫폼 변경형 골든 샘플 구조

주제 예: 카카오톡 지원 종료, 내 폰도 해당될까

필수 구조:

1. 영향 범위
   - 내 기기
   - 내 OS 버전
   - 내 계정
   - 내 결제 정보
   - 내 백업 가능 여부

2. 변경 전후 비교표
   - 변경 전 사용 가능 기능
   - 변경 후 제한 기능
   - 지원 종료일
   - 백업 가능 기간
   - 대체 방법

3. 지금 확인할 설정
   - 앱 버전
   - OS 버전
   - 백업 상태
   - 계정 로그인 수단
   - 결제 또는 구독 상태
   - 공식 공지 링크 후보

4. FAQ
   - 내 휴대폰도 지원 종료 대상인가요?
   - 지원 종료 전에 백업해야 할 것은 무엇인가요?
   - 대체 방법은 어디에서 확인하나요?

## 7. AI 업무형 골든 샘플 구조

주제 예: 크롬 AI 기능 켜기 전에 확인할 설정

필수 구조:

1. 기능 소개보다 업무 영향
   - 새 기능이 무엇인지보다 내 업무 흐름에서 무엇이 바뀌는지를 먼저 설명한다.

2. 누가 쓰면 좋은지
   - 반복 요약이 많은 사람
   - 자료 정리를 자주 하는 직장인
   - 브라우저 안에서 검색과 정리를 함께 하는 사람

3. 기존 업무 전후 비교
   - 기존: 검색, 복사, 요약, 정리, 검수 분리
   - 변경 후: 브라우저 안에서 요약과 정리 일부 자동화
   - 남는 일: 사실 확인, 문맥 판단, 보안 검토

4. 바로 써볼 프롬프트/설정
   - 이 페이지 내용을 3줄로 요약해줘.
   - 내가 확인해야 할 조건만 체크리스트로 정리해줘.
   - 표에 있는 날짜와 금액만 따로 뽑아줘.

5. 주의점
   - 회사 기밀 입력 금지
   - 자동 요약 결과 그대로 제출 금지
   - 출처와 날짜 재확인
   - 개인 설정과 데이터 사용 범위 확인

6. FAQ
   - AI 기능을 켜기 전에 어떤 설정을 먼저 봐야 하나요?
   - AI 요약 결과는 그대로 써도 되나요?
   - 업무에서 AI 기능을 쓸 때 가장 중요한 기준은 무엇인가요?

## 8. viral_issue_decode 골든 샘플 구조

주제 예: 넷플릭스 신작 반응이 갈린 이유, 시청자가 먼저 본 3가지

필수 구조:

1. 이슈 한 줄 요약
   - 지금 어떤 작품/경기/이슈가 화제인지 한 문장으로 정리한다. 배경 설명은 2~3줄을 넘기지 않는다.

2. 왜 사람들이 클릭하는가
   - 호기심/공감/논쟁/놀라움 중 어떤 반응 구조가 클릭을 만들었는지 설명한다.
   - 단순 "화제다"가 아니라 반응 구조를 분석한다.

3. 반응이 갈린 포인트 3개
   - 호평 측 근거
   - 비판 측 근거
   - 중간 독자가 주목한 포인트

4. 돈/플랫폼/팬덤/콘텐츠 구조 해석
   - 이 이슈가 팬덤 소비, OTT 전략, 플랫폼 알고리즘, 티켓팅 구조 중 어디에 연결되는지 해석한다.
   - 독자가 소비자로서 알아야 할 구조적 이유를 제시한다.

5. 독자가 볼 다음 포인트
   - 이 이슈가 앞으로 어떻게 전개될지, 독자가 다음에 확인할 것은 무엇인지 제시한다.

6. 관련 evergreen 내부링크 후보
   - 팬덤 소비 구조 글, OTT 요금제 비교 글, 티켓팅 방법 글 등 관련 evergreen 주제를 제안한다.

7. 안전한 FAQ
   - 이 이슈의 핵심을 묻는 3개 질문.
   - 루머/사생활 질문은 포함하지 않는다.
   - 예: 반응이 갈린 이유는 무엇인가요? / 이 이슈와 연결된 소비 구조는 무엇인가요? / 다음에 확인할 포인트는 무엇인가요?

## 9. 제목 규칙

좋은 제목 (생활 의사결정형):

- 고유가 피해지원금 신청 전 이것부터 확인하세요
- 세금 환급금 조회 전 홈택스에서 먼저 볼 3가지
- 환불 지연 때 소비자가 먼저 남겨야 할 증거
- 카카오톡 지원 종료, 내 폰도 해당될까
- 크롬 AI 기능 켜기 전에 확인할 설정

좋은 제목 (viral_issue_decode형):

- 넷플릭스 신작 반응이 갈린 이유, 시청자가 먼저 본 3가지
- 손흥민 경기 반응이 갈린 이유, 팬들이 본 핵심 장면

- OTT 요금제 변화, 시청자에게 미치는 실질 영향

나쁜 제목:

- 지원금 신청 전 이것부터 확인하세요
- 세금 환급 신청 전 이것부터 확인하세요
- 사람들이 놓친 이면
- 진짜 변수는 따로 있다
- 유행 뒤에 숨은 돈의 흐름
- 결국 누가 더 내나
- ○○ 충격 근황
- ○○ 결국 터졌다
- ○○ 소름 돋는 이유
- ○○ 사생활 논란 총정리

제목 판단 기준:

- 주제명이 구체적인가?
- 독자의 손해나 궁금증이 보이는가?
- 읽고 얻는 이득이 보이는가?

- 본문이 실제로 제목의 약속을 지키는가?
- (viral_issue_decode) 루머/사생활이 아닌 반응 구조 해석인가?

## 11. 자동화 편집장 체크리스트

발행 전 반드시 확인:

- 이 글의 타겟은 누구인가?
- 이 글의 한 문장 결론은 무엇인가?
- 독자가 검색할 질문 3개가 본문에 답변됐는가?
- 제목과 본문이 같은 약속을 하는가?
- 해시태그가 주제와 일치하는가?
- 다른 유형의 표현이 섞이지 않았는가?
- 읽고 나서 바로 할 행동이 있는가?

발행 차단 후보:

- fallback/test 후보가 publish 모드로 올라온 경우
- 세금환급형에 사용처, 바우처, 가맹점, 지역상품권 표현이 섞인 경우
- 지원금형인데 대상, 기간, 금액, 신청 방법, 사용처 중 핵심 정보가 거의 없는 경우
- 제목은 구체적인데 본문이 "공식 확인하세요" 반복으로 끝나는 경우
- FAQ가 주제명을 포함하지 않거나 다른 유형의 질문으로 구성된 경우
- 해시태그가 주제와 다른 유형을 섞는 경우
- viral_issue_decode인데 루머/사생활 폭로 중심으로 구성된 경우
- viral_issue_decode인데 evergreen 내부링크 후보가 없는 경우
- viral_issue_decode 제목에 "충격", "소름", "난리났다", "결국 터졌다" 등이 포함된 경우
